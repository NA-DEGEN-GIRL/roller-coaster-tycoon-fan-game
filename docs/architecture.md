# Architecture

## Runtime Stack

- Vite
- TypeScript
- Three.js

현재는 빠른 프로토타입을 위해 단일 엔트리 파일인 `src/main.ts`에 씬, 상태, 입력, 시뮬레이션을 함께 둔다. 기능이 늘어나면 다음 단위로 나누는 것이 좋다.

- `scene/`: renderer, camera, lights, resize
- `map/`: tile grid, height sampling, placement validation
- `prefabs/`: carousel, food stalls, flat rides
- `simulation/`: guests, path graph, ride demand
- `ui/`: toolbar, stats, status messages

## Tile Model

맵은 정수 좌표 `(x, z)`를 기준으로 한 정사각형 타일이다. 키는 `"x,z"` 문자열을 사용한다. 현재 프로토타입은 배치와 동선 검증을 쉽게 하기 위해 전체 평지로 둔다.

- `paths`: 길이 설치된 타일
- `queuePaths`: 대기줄 길이 설치된 타일
- `trees`: 나무가 설치된 타일
- `benches`: 일반 길 외곽에 붙은 벤치 장식
- `occupied`: 놀이기구 prefab이 점유한 타일
- `entrances`: 놀이기구 입구가 설치된 타일
- `exits`: 놀이기구 출구가 설치된 타일
- `rides`: 설치된 놀이기구 인스턴스
- `parkEntrance`: 공원 메인 입구 prefab과 연결된 출입 path

배치 검증은 타일 단위로 처리한다. 길과 대기줄은 빈 타일에만 놓을 수 있고, 회전목마는 3x3 footprint가 모두 비어 있어야 설치된다. 길 메시는 타일보다 아주 조금 크게 만들어 인접 길과 시각적으로 이어지도록 한다.

## Park Entrance

공원 메인 입구는 테스트 맵 왼쪽에 고정 배치된 prefab이다. 입구 prefab을 클릭하면 선택 패널이 나타나며 `Open`/`Close`를 바꿀 수 있다.

- `Open`: 일정 간격으로 새 손님이 입구 밖에서 생성되어 연결된 일반 길로 걸어 들어온다.
- `Close`: 새 손님 생성이 멈추고, 탑승 중이거나 탑승 애니메이션 중인 손님을 제외한 손님은 입구 path까지 BFS로 퇴장 경로를 잡는다.
- 퇴장 경로가 길 철거로 끊기면 손님은 기존 `seeking` 상태로 주변을 헤매다가 다시 가까운 일반 길을 찾는다. 입구까지 이어진 길이 없으면 퇴장하지 못한다.
- 테스트 맵이므로 시작 시점에도 이미 공원 안에 손님이 배치되어 있다.

## Economy

테스트맵은 큰 시작 예산을 가진다. 금액은 `src/gameConfig.ts`의 `gameConfig.economy`에서 조정한다.

- `startingCash`: 시작 현금
- `parkAdmissionFee`: 공원 입장료 기본값
- `rideDefaults.carousel`: 회전목마 탑승료, 흥미도, 강도, 멀미, 인기도 기본값
- `guestDecision`: 입장료/탑승료 판단, 줄 인내심, 거절 쿨다운 상수
- `buildCosts`: 길, 대기줄 길, 회전목마, 입구/출구, 나무, 벤치 설치 비용
- `refundRates`: 철거 시 환급 비율

초기 테스트맵 시드 배치는 비용을 차감하지 않는다. 사용자가 직접 건설하는 경우에만 비용을 차감하고, 현금이 부족하면 설치하지 않는다. 나무 비용은 현재 크기 배율에 따라 증가한다.
공원 입장료와 놀이기구 탑승료는 실행 중에도 UI에서 수정할 수 있다. 공원 입구를 선택하면 입장료를, 놀이기구를 선택하면 해당 놀이기구 탑승료를 수정한다. 사이드바의 경제 설정 패널에서는 입장료와 설치된 놀이기구 가격을 한 번에 볼 수 있다.

손님은 입장 시 보유 현금, 가격 민감도, 입장 가치 점수를 보고 공원에 들어올지 판단한다. 입장하면 입장료가 손님 돈에서 빠지고 `parkCash`에 더해진다. 놀이기구 탑승료는 대기줄 진입이 아니라 실제 `boarding` 시작 시점에 차감된다.

## Ride Placement

건설 UI는 카탈로그 팝업 형태로 나뉜다. 건설 패널에는 `기본 도구`, `길`, `놀이기구`, `꾸미기` 탭만 노출하고, 각 탭을 누르면 해당 항목 팝업이 열린다. 일반 길과 대기줄 길은 `길` 팝업에 있고, 회전목마는 `놀이기구 > 잔잔한 놀이기구` 항목이다. 현재 꾸미기 항목에는 전나무, 벚꽃 나무, 벤치가 들어 있다.

회전목마는 설치 후 선택 상태가 된다. 선택된 회전목마에 대해 다음 작업을 할 수 있다.

- 기본 `Select` 툴에서 회전목마 footprint, 입구, 출구를 클릭해 놀이기구를 선택
- 철거 툴을 제외한 다른 설치 툴에서도 기존 놀이기구나 게이트를 클릭하면 자동으로 `Select` 툴로 전환해 선택
- 선택 패널의 `Open`/`Closed` 버튼으로 운영 상태 선택
- 선택된 놀이기구가 없으면 선택 패널은 DOM에서 `hidden` 상태가 되어 UI에서 보이지 않는다.
- 선택 패널의 입구/출구 버튼으로 해당 놀이기구에 종속된 게이트 설치
- 대기줄 연결
- 출구를 일반 길에 연결

회전목마 prefab을 새로 설치하면 자동으로 입구 설치 툴로 전환된다. 입구 설치가 끝나면 출구 설치 툴로 이어지며, 사용자는 언제든 다른 메인 툴을 눌러 이 흐름을 중단할 수 있다.
회전목마 설치 툴에서는 마우스를 올린 타일에 3x3 footprint와 간단한 3D prefab 프리뷰를 표시한다. `R` 키를 누르면 설치 방향이 90도씩 바뀌며, 설치된 회전목마의 전면 표시와 운영 패널도 같은 방향으로 돌아간다. 현재는 RCT처럼 4방향 회전만 지원한다.

회전목마가 실제로 동작하려면 open 상태이면서 입구, 출구, 입구와 연결된 대기줄, 출구와 연결된 일반 길이 모두 있어야 한다. 이 조건을 만족하지 않으면 상태 표시등은 closed 색으로 남고 회전 애니메이션도 멈춘다.
선택 패널은 입구 대기줄 연결과 tail의 일반 길 연결을 분리해서 표시한다. 대기줄이 입구까지 이어져도 손님이 들어갈 일반 길이 tail 쪽에 없으면 `Connect queue tail to path` 상태가 표시된다.

입구 타일은 대기줄 바닥 재질을, 출구 타일은 일반 길 바닥 재질을 사용한다. 둘 다 일반 길 타일과 같은 크기로 만들어 주변 동선과 시각적으로 이어지게 한다. 게이트는 배치된 타일에서 인접한 놀이기구 footprint 방향을 향하도록 자동 회전한다.

입구/출구 게이트는 bar gate 애니메이션을 가진다. 입구 bar는 `boarding` 손님이 입구를 지나는 동안 열리고, 출구 bar는 `unloading` 및 `exiting` 손님이 출구를 지나는 동안 열린다. 하차 손님은 출구에 연결된 일반 길에서 생성되지 않고, 출구 게이트의 놀이기구 안쪽 경계에서 시작해 바깥 길로 걸어 나온다.

## Guest Simulation

손님은 일반 길과 대기줄 타일 위에서 이동한다. 각 손님은 `id`, `from`, `to`, `progress`, `speed`, `pause`, `state` 값을 가지며, UI 표시용으로 `money`, `hunger`, `tiredness`, `happiness`, `nausea`를 함께 가진다. 행동 판단용으로 `intent`, `targetPathKey`, `targetRideId`, `priceSensitivity`, `patience`, `ridePreference`, `visitedRideIds`, `rejectedRideIds`, `thought`를 가진다.

- 일반 이동은 일반 길 graph에서만 처리한다. 대기줄은 일반 길 graph와 섞지 않는다.
- 일반 이동은 목적지를 고른 뒤 일반 길 graph의 BFS route를 따라간다. 산책도 임의의 일반 길 타일을 목적지로 잡고 이동한다.
- 공원이 open이면 새 손님은 공원 메인 입구 밖에서 일반 길로 입장한다.
- 공원이 close이면 손님은 일반 길 graph에서 공원 입구까지의 최단 경로로 이동해 맵에서 제거된다.
- 손님은 open 상태이고 연결이 완료된 놀이기구만 목적지 후보로 본다.
- 놀이기구 선택 점수는 흥미도, 인기도, 손님 취향에서 가치를 만들고, 탑승료, 줄 길이, 멀미 부담을 차감한다.
- 점수가 낮거나 돈이 부족하거나 줄이 가득 차면 손님은 해당 놀이기구를 잠시 거절 목록에 넣고 다른 목적지를 고른다.
- 손님은 일반 길에서 사용자가 지정한 대기줄 entry와 인접한 타일에 도착했을 때만 대기줄 입장을 시도한다.
- 대기줄 내부는 놀이기구 입구와 연결된 별도 graph로 관리한다.
- 대기줄 타일은 방향 버튼을 사용하지 않는다. 입구 옆에서 첫 대기줄을 놓고, 그 다음부터 이전 대기줄 옆에 이어 깔면 설치 순서대로 `nextKey`가 입구 쪽 타일을 가리킨다.
- 대기줄 펜스는 각 타일 중심선 연결을 보고 자동 재계산한다. 입구 쪽 다음 타일, 뒤에서 이어지는 대기줄, 사용자가 지정한 일반 길 entry 변만 열고 나머지는 막는다.
- 일반 길에서 대기줄로 들어갈 수 있는 지점은 현재 tail 타일에 사용자가 연결한 `entryPathKey` 하나다. `Queue` 툴로 tail에 닿은 일반 길 타일을 클릭하면 그 길이 entry로 지정되고, 다른 인접 일반 길을 다시 클릭하면 entry가 이동한다. tail 뒤에 대기줄을 더 이어 깔면 이전 tail의 entry 지정은 무효화되고 새 tail에서 다시 지정한다.
- 선택된 entry 변에는 노란 진입 화살표를 표시한다. 대기줄 내부 타일에는 기능성 방향 화살표를 표시하지 않는다.
- 길이 철거되어 목표가 사라지면 `seeking` 상태로 주변을 헤매다가 가까운 일반 길로 복귀한다.
- open 상태의 놀이기구 입구와 연결된 대기줄 tail에 도착하면 `queueing` 상태로 대기줄 안을 걸어 들어간다.
- 목표 queue 슬롯에 도착하면 `waiting` 상태로 멈춘다.
- 대기줄 타일은 현재 4명까지 수용하며, 타일 내부 슬롯은 앞뒤 타일과 같은 간격이 이어지도록 균등 배치한다.
- 탑승으로 앞쪽 슬롯이 비면 `waiting` 손님뿐 아니라 이미 줄로 걸어 들어가던 `queueing` 손님도 목표 슬롯을 다시 배정받고 앞 슬롯으로 걸어간다.
- 대기줄 슬롯이 모두 차 있으면 손님은 탑승하지 않고 다른 경로로 이동한다.
- 놀이기구가 `loading` 상태일 때 탑승 가능 인원만큼 대기 손님이 `boarding` 상태로 전환된다.
- `boarding` 손님은 일반 보행 속도로 입구를 지나 안쪽까지 걸어간 뒤 `riding` 상태가 되고 화면에서 숨겨진다.
- 운행 종료 후 `unloading` 상태가 끝나면 출구와 연결된 일반 길로 나온다.
- 철거 툴이 아닌 상태에서 손님 mesh를 클릭하면 자동으로 `Select` 툴로 전환되고, 작은 손님 팝업에 현재 행동과 상태 수치를 표시한다.
- 놀이기구, 게이트, 공원 입구, 나무, 벤치는 바닥 타일 추정보다 물체 raycast를 우선해 클릭한다.
- 전나무와 벚꽃 나무는 타일을 단독 점유하지 않고 빈 잔디 타일 안에 최대 4그루까지 자유 위치로 배치한다. 길, 대기줄, 놀이기구, 입구/출구 타일과는 겹치지 않으며, 나무끼리도 크기에 따른 최소 간격을 둔다. 설치 전에는 실제 나무 형태의 반투명 프리뷰를 표시하고, 꾸미기 팝업의 크기 슬라이더 또는 `[`/`]`로 크기를, `R`로 4방향 회전값을 정한다.
- 벤치는 일반 길 타일의 외곽 edge에만 설치된다. 벤치는 `paths` graph를 바꾸지 않고 별도 `benches` 맵에 저장되므로 손님 이동 경로를 막지 않는다. 마우스 위치에 가까운 길 edge를 기준으로 미리보기를 보여주며, 인접 edge가 다른 길/대기줄/입구/출구/놀이기구/나무와 맞닿아 있으면 설치하지 않는다.
- 손님 팝업의 follow 버튼을 켜면 카메라 target이 선택 손님 위치를 부드럽게 따라간다. WASD 이동이나 휠 클릭 드래그로 수동 이동을 시작하면 follow를 끈다.

## Camera Controls

카메라는 orthographic isometric view를 유지한다.

- 마우스 휠: 확대/축소
- `Q`: 왼쪽 쿼터뷰 회전
- `E`: 오른쪽 쿼터뷰 회전
- `WASD`: 현재 화면 방향 기준 카메라 이동

`Path`와 `Queue` 툴은 마우스를 누른 채 타일 위로 드래그하면 연속 설치된다. `Bulldoze` 툴도 드래그 삭제를 지원하지만, 첫 클릭으로 삭제한 대상 종류만 같은 드래그에서 계속 삭제한다. 예를 들어 첫 대상이 일반 길이면 일반 길만 삭제하고, 놀이기구나 나무는 같은 드래그에서 삭제하지 않는다.

Camera 패널의 `Continuous QE` 옵션을 켜면 `Q/E`가 누르고 있는 동안 연속 회전한다. 옵션을 끄는 순간 현재 각도에서 가장 가까운 90도 쿼터뷰로 스냅한다.

## Localization

UI 기본 언어는 한국어다. 사이드 패널의 언어 선택에서 English로 전환할 수 있으며, 고정 UI 문구와 주요 상태 메시지, 선택 패널, 손님 팝업, 디버그 상태 줄이 현재 언어로 갱신된다.

## Prefab Rides

회전목마는 첫 prefab 구현이다.

- 3x3 footprint
- 회전목마 전용 3x3 foundation 바닥과 낮은 경계 레일을 가진다. footprint 바닥 표현은 놀이기구 prefab마다 다르게 정의한다.
- 고정 base와 회전하는 rotor 분리
- 지붕은 회전목마 전용 빨강/크림색 우산형 canopy segment로 구성한다.
- 말 좌석은 rotor의 자식으로 두고 애니메이션 루프에서 회전한다.
- 탑승한 손님 수를 기준으로 회전목마 좌석에 rider visual을 표시한다. 만석이면 모든 좌석이 차고, 일부 탑승이면 비율에 맞춰 랜덤 좌석이 켜진다.
- 운행 상태는 `idle -> loading -> running -> unloading` 순서로 전환된다.
- 회전목마 음악은 Web Audio API 합성 프리셋과 실제 오디오 asset 프리셋을 함께 지원한다. 선택 패널의 음악 on/off와 놀이기구 open 상태를 모두 만족할 때 들린다.
- 회전목마 음악은 `fairgroundOrgan`, `waltzOrgan`, `musicBox`, `militaryBandOrgan` 프리셋 중에서 놀이기구별로 선택한다. 기본값은 RCT풍에 가까운 페어그라운드 오르간이다.
- 음악 볼륨은 `cameraTarget`과 회전목마 중심 사이의 거리로 계산한다. 가까우면 선명하고 멀어질수록 fade out한다.
- 회전목마 음악은 줌아웃할수록 추가로 작아진다.
- 브라우저 autoplay 정책 때문에 첫 pointer/key 입력에서 `AudioContext`를 생성/재개한다. 일시정지 중에는 master gain을 0으로 줄인다.
- 운행 중에는 rotor가 회전하고, 각 말/탑승자 묶음은 고정된 폴을 따라 위아래로 움직인다.

## Park Ambience

- 군중 ambience는 합성하지 않고 `public/audio/crowd/`의 실제 오디오 루프를 Web Audio로 믹싱한다.
- 볼륨은 `cameraTarget` 주변의 보이는 손님 밀도, 오디오 테스트 존 보정값, `cameraZoom`을 기준으로 계산한다. 근처 손님이 많을수록 커지며 줌아웃하거나 손님이 멀면 작아진다.
- 웃음, 아이 울음, 놀이기구 반응은 짧은 one-shot asset으로 관리하고, 군중 밀도와 랜덤 간격에 따라 재생한다.
- `loading` 상태는 탑승 인원 선택과 탑승 보행 애니메이션을 함께 관리한다.
- `running` 상태에서만 rotor가 회전한다.

이 패턴을 유지하면 바이킹, 관람차, 푸드 스탠드 같은 다음 세트를 같은 방식으로 추가할 수 있다.
